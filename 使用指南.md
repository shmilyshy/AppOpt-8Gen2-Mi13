# 骁龙8Gen2小米13线程优化模块 - 使用指南

## 快速开始

### 1. 安装模块

```bash
# 在Magisk Manager中
模块 -> 从本地安装 -> 选择zip文件 -> 重启
```

### 2. 验证安装

```bash
# 重启后在终端运行
cat /data/local/tmp/appopt_8gen2.log
```

应该看到：
```
✓ 检测到小米13 (fuxi)，骁龙8Gen2优化已启用
设备型号: 2211133C
处理器: SM8550
CPU架构: 0-2小核 | 3-6中大核 | 7超大核
```

## 配置文件说明

### 位置
```
/data/adb/modules/AppOpt_8Gen2_Mi13/applist.conf
```

### 基本语法

```bash
# 1. 绑定整个应用到指定核心
包名=CPU核心范围

# 2. 绑定特定线程到指定核心
包名{线程名}=CPU核心范围

# 3. 使用通配符
包名{线程名*}=CPU核心范围
```

### 核心范围说明

根据你的小米13实际配置：

| 核心范围 | 频率 | 适用场景 |
|---------|------|---------|
| 0-2 | 2016MHz | 后台任务、低功耗应用 |
| 3-4 | 2803MHz | 日常应用渲染 |
| 5-6 | 2803MHz | 游戏渲染、视频处理 |
| 7 | 3187MHz | 游戏主逻辑、极限性能 |
| 0-3 | 混合 | 轻度游戏主进程 |
| 6-7 | 高性能 | 重度游戏渲染 |

## 实用示例

### 游戏优化

#### 王者荣耀（高帧率）
```bash
com.tencent.tmgp.sgame=0-2
com.tencent.tmgp.sgame{UnityMain*}=6-7
```
效果：主进程省电，渲染流畅

#### 原神（高画质）
```bash
com.miHoYo.Yuanshen=0-3
com.miHoYo.Yuanshen{UnityMain}=6-7
com.miHoYo.Yuanshen{UnityGfxDeviceW}=6-7
```
效果：保证帧率稳定

#### 和平精英（竞技）
```bash
com.tencent.tmgp.pubgmhd=0-3
com.tencent.tmgp.pubgmhd{RHIThread}=6-7
com.tencent.tmgp.pubgmhd{*Thread}=4-7
```
效果：降低延迟，提升响应

### 应用优化

#### 微信（省电）
```bash
com.tencent.mm=0-2
com.tencent.mm{RenderThread}=3-4
```
效果：日常聊天省电，滑动流畅

#### 微信小程序（性能）
```bash
com.tencent.mm:appbrand0=0-3
com.tencent.mm:appbrand0{*Re}=6-7
```
效果：小程序运行流畅

#### 抖音（视频流畅）
```bash
com.ss.android.ugc.aweme=0-3
com.ss.android.ugc.aweme{RenderThread}=5-6
```
效果：视频播放流畅，不卡顿

## 常见线程名称

### Unity引擎游戏
```
UnityMain          # Unity主线程 -> 建议绑定6-7
UnityGfxDeviceW    # 图形设备线程 -> 建议绑定6-7
Job.Worker         # 任务工作线程 -> 建议绑定0-3
UnityChoreographer # 编排线程 -> 建议绑定3-4
```

### UE引擎游戏
```
GameThread      # 游戏逻辑 -> 建议绑定6-7
RenderThread    # 渲染线程 -> 建议绑定6-7
RHIThread       # 渲染接口 -> 建议绑定3-4
```

### 普通应用
```
RenderThread    # 渲染线程 -> 建议绑定3-4
MainThread      # 主线程 -> 建议绑定0-2
```

## 调优技巧

### 1. 游戏卡顿
如果游戏卡顿，尝试：
```bash
# 增加渲染线程核心范围
游戏包名{RenderThread*}=5-7  # 原来可能是6-7
```

### 2. 游戏发热
如果游戏发热严重，尝试：
```bash
# 减少使用超大核
游戏包名{UnityMain}=5-6  # 原来可能是6-7
```

### 3. 应用耗电
如果应用耗电，尝试：
```bash
# 限制到小核
应用包名=0-2
应用包名{RenderThread}=3-4
```

### 4. 视频播放卡顿
如果视频卡顿，尝试：
```bash
# 增加渲染核心
视频应用{RenderThread}=5-6  # 原来可能是3-4
```

## 获取应用包名

### 方法1：使用MT管理器
1. 打开MT管理器
2. 点击左上角菜单
3. 选择"应用管理"
4. 找到应用，长按查看详情
5. 复制包名

### 方法2：使用终端
```bash
# 列出所有应用
pm list packages

# 搜索特定应用（例如微信）
pm list packages | grep tencent

# 查看正在运行的应用
ps -A | grep -v "system"
```

### 方法3：使用ADB
```bash
# 电脑连接手机后
adb shell pm list packages
```

## 实时监控

### 查看CPU使用情况
```bash
# 查看各核心频率
cat /sys/devices/system/cpu/cpu*/cpufreq/scaling_cur_freq

# 查看CPU占用
top -m 10
```

### 查看应用线程
```bash
# 查看应用的所有线程
ps -T -p $(pidof 包名)

# 例如查看微信
ps -T -p $(pidof com.tencent.mm)
```

### 查看线程绑定
```bash
# 查看线程绑定的CPU
cat /proc/线程ID/status | grep Cpus_allowed_list
```

## 高级技巧

### 1. 针对不同场景创建多个配置

创建多个配置文件：
```bash
applist.conf.performance  # 性能模式
applist.conf.balanced     # 均衡模式
applist.conf.powersave    # 省电模式
```

切换配置：
```bash
cp applist.conf.performance applist.conf
```

### 2. 临时禁用优化
```bash
# 停止AppOpt进程
killall AppOpt

# 恢复
sh /data/adb/modules/AppOpt_8Gen2_Mi13/service.sh
```

### 3. 备份配置
```bash
# 备份当前配置
cp /data/adb/modules/AppOpt_8Gen2_Mi13/applist.conf /sdcard/applist.conf.backup

# 恢复配置
cp /sdcard/applist.conf.backup /data/adb/modules/AppOpt_8Gen2_Mi13/applist.conf
```

## 故障排除

### 问题1：模块安装后无效果
```bash
# 检查模块是否启用
ls /data/adb/modules/

# 检查进程
ps -A | grep AppOpt

# 查看日志
cat /data/local/tmp/appopt_8gen2.log
```

### 问题2：某个应用优化无效
```bash
# 1. 确认包名正确
pm list packages | grep 关键词

# 2. 确认线程名正确
ps -T -p $(pidof 包名)

# 3. 检查配置文件语法
cat /data/adb/modules/AppOpt_8Gen2_Mi13/applist.conf | grep 包名
```

### 问题3：系统卡顿
```bash
# 临时禁用模块
killall AppOpt

# 如果恢复正常，说明配置有问题
# 恢复默认配置
cp /data/adb/modules/AppOpt_8Gen2_Mi13/applist.conf.example applist.conf
```

- 游戏主进程：0-2
- 游戏渲染：3-6
