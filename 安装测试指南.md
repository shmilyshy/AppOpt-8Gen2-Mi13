## 🚀 安装步骤

### 1. 准备工作
- 确保已安装 Magisk 或 KernelSU
- 确保设备是小米13 (2211133C/fuxi)
- 备份重要数据

### 2. 刷入模块
```bash
# 方法1: Magisk Manager
1. 打开 Magisk Manager
2. 点击 "模块" 标签
3. 点击 "从本地安装"
4. 选择 AppOpt_8Gen2_Mi13_v1.1_刘十三.zip
5. 等待安装完成
6. 重启设备

# 方法2: ADB命令
adb push AppOpt_8Gen2_Mi13_v1.1_刘十三.zip /sdcard/
adb shell su -c "magisk --install-module /sdcard/AppOpt_8Gen2_Mi13_v1.1_刘十三.zip"
adb reboot
```

### 3. 验证安装
重启后检查:
```bash
# 1. 查看模块状态
# Magisk Manager → 模块 → 查看模块名称
# 应该显示: ✅ 骁龙8Gen2小米13线程优化

# 2. 查看日志
adb shell su -c "cat /data/local/tmp/appopt_8gen2.log"

# 应该看到:
# ✓ 检测到小米13 (fuxi)，骁龙8Gen2优化已启用
# 设备型号: 2211133C
# 处理器: SM8550
# CPU架构: 0-2小核 | 3-6中大核 | 7超大核
# AppOpt进程ID: xxxxx
# ✅ 优化已生效

# 3. 检查进程
adb shell su -c "ps -A | grep AppOpt"
# 应该看到 AppOpt 进程在运行
```

## ⚙️ 配置修改

### 查看当前配置
```bash
adb shell su -c "cat /data/adb/modules/AppOpt_8Gen2_Mi13/applist.conf | head -20"
```

### 修改配置
```bash
# 1. 拉取配置文件
adb pull /data/adb/modules/AppOpt_8Gen2_Mi13/applist.conf

# 2. 编辑配置 (使用文本编辑器)
# 格式: 包名=CPU核心范围
# 例如: com.tencent.mm=0-2

# 3. 推送回设备
adb push applist.conf /data/adb/modules/AppOpt_8Gen2_Mi13/

# 4. 重启AppOpt (配置即时生效)
adb shell su -c "killall AppOpt"
adb shell su -c "/data/adb/modules/AppOpt_8Gen2_Mi13/AppOpt &"
```

### 配置示例
```conf
# 游戏优化 - 主进程小核，渲染大核
com.tencent.tmgp.sgame=0-2
com.tencent.tmgp.sgame{UnityMain*}=6-7

# 日常应用 - 主进程小核，渲染中核
com.tencent.mm=0-2
com.tencent.mm{RenderThread}=3-4

# 省电模式 - 全部小核
com.某应用=0-2
```

## 🔧 故障排查

### 问题1: 模块未生效
```bash
# 检查模块状态
adb shell su -c "ls -la /data/adb/modules/AppOpt_8Gen2_Mi13/"

# 查看日志
adb shell su -c "cat /data/local/tmp/appopt_8gen2.log"

# 手动启动
adb shell su -c "/data/adb/modules/AppOpt_8Gen2_Mi13/service.sh"
```

### 问题2: AppOpt进程未运行
```bash
# 检查文件是否存在
adb shell su -c "ls -la /data/adb/modules/AppOpt_8Gen2_Mi13/AppOpt"

# 检查权限
adb shell su -c "chmod 755 /data/adb/modules/AppOpt_8Gen2_Mi13/AppOpt"

# 手动启动
adb shell su -c "/data/adb/modules/AppOpt_8Gen2_Mi13/AppOpt &"
```

### 问题3: 线程绑定未生效
```bash
# 1. 确认AppOpt运行
ps -A | grep AppOpt

# 2. 确认配置文件正确
cat /data/adb/modules/AppOpt_8Gen2_Mi13/applist.conf | grep 应用包名

# 3. 重启应用
am force-stop 应用包名
# 然后重新打开应用

# 4. 再次检查绑定
cat /proc/$(pidof 应用包名)/status | grep Cpus_allowed_list
```

### 问题4: 与Scene冲突
```bash
# 查看日志中的Scene检测
cat /data/local/tmp/appopt_8gen2.log | grep Scene

# 应该看到:
# ⚠ 检测到Scene模块，建议配置兼容模式
# 已延迟启动以兼容Scene

# Scene和AppOpt可以共存:
# - Scene负责CPU调度策略
# - AppOpt负责线程亲和性
# - 两者互补，不冲突
```
