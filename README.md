# 骁龙8Gen2 小米13专用线程优化模块

## 模块简介

本模块专为**小米13标准版（2211133C / fuxi）**优化，基于实际设备测试数据，通过精确的CPU线程亲和性绑定，提升游戏性能和日常使用流畅度。

## 设备要求

- **设备型号**: 小米13标准版 (2211133C)
- **处理器**: 骁龙8 Gen2 (SM8550)
- **系统**: Android 12+ / MIUI 14+
- **必需**: Magisk v20.4+ 已安装并正常运行

⚠️ **重要**: 本模块需要 Magisk，如果你还没安装，请先安装 Magisk！

## CPU架构说明（实测数据）

根据你的设备实际检测：

- **CPU 0-2**: 效能小核 (307-2016 MHz) - 后台任务和低功耗场景
- **CPU 3-6**: 性能中大核 (499-2803 MHz) - 中高负载任务
- **CPU 7**: 超大核 (595-3187 MHz) - 极限性能需求

调度器: **walt** (高通自研调度器)

## 优化策略

### 游戏优化
- 主进程绑定到小核和部分中核（0-3），降低功耗
- 渲染线程（UnityMain、RenderThread等）绑定到中大核+超大核（6-7），保证帧率
- 游戏逻辑线程（GameThread）绑定到超大核（7），保证响应速度

### 应用优化
- 日常应用主进程绑定小核（0-2），节省电量
- 渲染线程绑定中核（3-4），保证流畅度
- 视频类应用渲染线程可使用大核（5-6），保证播放流畅

## 安装方法

### 前置条件检查

1. **检查 Magisk 是否已安装**
```bash
# 在 Termux 或 ADB Shell 中运行
magisk --version
```
如果显示版本号，说明已安装。如果提示命令不存在，需要先安装 Magisk。

2. **确认设备型号**
```bash
getprop ro.product.model
# 应该显示: 2211133C
```

### 安装步骤

1. 确保已安装 **Magisk v20.4+**
2. 在 Magisk Manager 中选择"模块"
3. 点击"从本地安装"
4. 选择本模块的 zip 文件
5. 等待安装完成
6. 重启设备
7. 模块会自动检测设备型号并启用优化

### 安装后验证

重启后，检查模块是否正常运行：
```bash
# 查看日志
cat /data/local/tmp/appopt_8gen2.log

# 检查进程
ps -A | grep AppOpt
```

## 配置文件

配置文件路径：`/data/adb/modules/AppOpt_8Gen2_Mi13/applist.conf`

### 规则格式

```
# 基础规则：将整个应用绑定到指定核心
包名=CPU核心范围

# 线程规则：将特定线程绑定到指定核心
包名{线程名}=CPU核心范围
```

### 示例

```bash
# 王者荣耀 - 主进程小核，渲染超大核
com.tencent.tmgp.sgame=0-2
com.tencent.tmgp.sgame{UnityMain*}=6-7

# 微信 - 主进程小核，渲染中核
com.tencent.mm=0-2
com.tencent.mm{RenderThread}=3-4
```

## 特殊功能

### 自动禁用MIUI性能服务
模块会自动禁用小米的 `joyose` 性能服务（已检测到你设备上有此服务），避免冲突。

### 禁用高通IRQ均衡
禁用 `msm_irqbalance` 服务（已检测到正在运行），由模块接管CPU调度。

### CPU核心全开
确保所有CPU核心在需要时都能被调用。

### 设备验证
模块会在启动时验证设备型号，确保是小米13才启用优化。

## 常见问题

**Q: 为什么安装后没有效果？**  
A: 请先确认已安装 Magisk，然后查看日志文件确认模块是否正常启动。

**Q: 我还没安装 Magisk，怎么办？**  
A: 请先安装 Magisk v20.4+，这是模块运行的必要条件。

**Q: 可以用在其他骁龙8 Gen2设备上吗？**  
A: 理论上可以，但本模块专为小米13优化，其他设备效果可能不同。

**Q: 修改配置后需要重启吗？**  
A: 不需要，修改后即时生效。

**Q: 会影响续航吗？**  
A: 合理的线程绑定反而能降低功耗，因为避免了不必要的核心迁移。

**Q: 为什么CPU核心总数显示5？**  
A: 这是系统显示问题，实际是8核心，不影响使用。

**Q: 我安装了 Scene，会冲突吗？**  
A: 不会冲突。本模块已自动检测 Scene 并延迟启动。建议 Scene 使用性能/均衡模式，核心数量全开。详见 `Scene兼容说明.md`。

**Q: Scene 和本模块应该用哪个？**  
A: 建议两个都用。Scene 负责 CPU 调度策略，本模块负责线程绑定，互补效果更好。

## 性能建议

根据你的设备配置：

1. **游戏时**: 建议开启性能模式，配合本模块效果最佳

## 技术原理

通过 Linux 的 CPU affinity 机制，将应用的特定线程绑定到指定的CPU核心上，减少线程在不同核心间的迁移开销，提升性能和能效比。

配合高通 walt 调度器，针对骁龙8 Gen2 的 3+4+1 核心架构进行深度优化。
